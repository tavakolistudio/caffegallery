# پرامپت نهایی برای Claude Code
## پروژه: منوی آنلاین غذای کوروش مال

یک وب‌سایت منوی دیجیتال حرفه‌ای، سریع و کاملاً ریسپانسیو برای مجموعه غذای کوروش مال طراحی و پیاده‌سازی کن.

این پروژه سه برند مستقل دارد:

1. **کافه گالری کوروش مال**
2. **سوشی گالری**
3. **پاستا گالری**

اطلاعات و قیمت‌های نهایی در فایل زیر قرار دارند:

```text
src/data/menu-data.ts
```

فایل داده تنها منبع حقیقت پروژه است. هیچ نام، قیمت، دسته‌بندی یا تعداد آیتمی را از روی حدس تغییر نده.

---

## 1. فناوری الزامی

پروژه را با این ابزارها بساز:

- Next.js 15 با App Router
- TypeScript با حالت strict
- Tailwind CSS
- Framer Motion فقط برای انیمیشن‌های محدود و ظریف
- Lucide React برای آیکن‌ها
- next/image برای تصاویر
- ESLint
- بدون دیتابیس و بدون بک‌اند در فاز اول
- داده‌ها فقط از `src/data/menu-data.ts` خوانده شوند

کد باید production-ready، ماژولار و قابل نگهداری باشد.

---

## 2. قانون حیاتی قیمت‌ها

قیمت‌ها در فایل داده به **تومان** و به‌صورت عدد صحیح ذخیره شده‌اند.

قوانین غیرقابل مذاکره:

- هیچ قیمتی را گرد نکن.
- هیچ صفر اضافه یا کم نکن.
- تومان را به ریال تبدیل نکن.
- قیمت‌های غیررُند سوشی مانند `889000`، `899000`، `879000`، `891000` و `959000` دقیقاً حفظ شوند.
- قیمت را فقط با تابع `formatToman` نمایش بده.
- نمایش نمونه صحیح: `۹۹۵٬۰۰۰ تومان`
- نمایش نمونه غلط: `۹۹۵ تومان` یا `۹٬۹۵۰٬۰۰۰ ریال`
- مقدار اصلی عددی باقی بماند؛ قیمت فرمت‌شده را داخل داده ذخیره نکن.
- تست خودکار برای صحت تعداد آیتم‌ها و قیمت‌های شاخص بنویس.

تعداد مورد انتظار:

- کافه گالری: **47 آیتم**
- سوشی گالری: **95 آیتم**
- پاستا گالری: **20 آیتم**
- مجموع: **162 آیتم**

در منوی چاپی، «آیس ماچا لاته بوبا – ۲۲۰٬۰۰۰ تومان» در دو صفحه تکرار شده بود و در دیتای آنلاین فقط یک بار قرار گرفته است.

در منوی چاپی دو آیتم با عنوان یکسان «آیس تی بوبا» و قیمت‌های متفاوت وجود دارند. در فایل داده با `variant` به‌صورت «مدل آبی» و «مدل نارنجی» تفکیک شده‌اند. نام و قیمت را تغییر نده.

## 3. زبان و جهت صفحه

- زبان اصلی و تنها زبان فاز اول: فارسی
- روی عنصر `html` از `lang="fa"` و `dir="rtl"` استفاده کن.
- فونت اصلی: Vazirmatn
- اعداد قیمت با رقم فارسی نمایش داده شوند.
- متن‌ها خوانا، فاصله خطوط مناسب و کاملاً RTL باشند.
- از فونت‌های تزئینی ناخوانا استفاده نکن.

## 4. ساختار صفحات

### صفحه اصلی `/`

صفحه اصلی باید شامل این بخش‌ها باشد:

- هدر مینیمال با نام «منوی غذای کوروش مال»
- Hero شیک و کوتاه، بدون متن تبلیغاتی اغراق‌آمیز
- سه کارت بزرگ برای ورود به:
  - کافه گالری کوروش مال
  - سوشی گالری
  - پاستا گالری
- هر کارت دارای عنوان، توضیح کوتاه، تعداد آیتم‌ها و رنگ هویت مخصوص همان برند باشد.
- عبارت ثابت: «تمام قیمت‌ها به تومان است.»
- فوتر ساده با فضاهای تنظیم‌شونده برای آدرس، شماره تماس و شبکه‌های اجتماعی
- اطلاعات تماس را حدس نزن؛ در `src/config/site.ts` مقدار خالی یا TODO قرار بده.

### مسیرهای برند

- `/cafe`
- `/sushi`
- `/pasta`

هر صفحه برند باید این بخش‌ها را داشته باشد:

1. هدر برند و دکمه بازگشت به صفحه اصلی
2. کادر جست‌وجوی سریع
3. نوار چسبان و افقی دسته‌بندی‌ها
4. گزینه «همه»
5. نمایش تعداد نتیجه
6. کارت‌های منو
7. ناوبری پایین صفحه در موبایل برای جابه‌جایی بین سه برند

## 5. طراحی بصری

طراحی باید شیک، مدرن، مینیمال، حرفه‌ای و مناسب اسکن QR داخل رستوران باشد.

- بدون شلوغی، افکت گیمینگ، ذرات زیاد یا انیمیشن آزاردهنده

## 6. کارت آیتم منو

هر کارت باید این موارد را نمایش دهد:

- نام آیتم
- `variant` در صورت وجود
- توضیحات یا ترکیبات در صورت وجود
- مقدار مانند «۱۶ تکه» یا «۲۵۰ گرم» در صورت وجود
- قیمت برجسته و خوانا
- نام دسته‌بندی به شکل کوچک و ظریف

قوانین:

- هیچ توضیح ساختگی تولید نکن.
- هیچ برچسب «پرفروش»، «پیشنهاد سرآشپز» یا تخفیف اضافه نکن.
- آیتم بدون تصویر نباید broken image داشته باشد.
- در نبود تصویر از یک fallback حرفه‌ای و ساده بر اساس رنگ برند استفاده کن.
- امکان اضافه کردن تصویر در آینده از طریق فیلد اختیاری `image` در داده وجود داشته باشد.
- قیمت از نام غذا واضح‌تر یا هم‌سطح آن باشد و هرگز کوچک و گم‌شده نباشد.

## 7. جست‌وجو و فیلتر

- جست‌وجو باید روی نام، دسته‌بندی، توضیحات و variant انجام شود.
- جست‌وجوی فارسی باید فاصله اضافی و نیم‌فاصله را تا حد ممکن normalize کند.

## 12. تست‌های اجباری

تست‌هایی بنویس که حداقل این موارد را بررسی کنند:

```ts
expect(cafeItems).toHaveLength(47);
expect(sushiItems).toHaveLength(95);
expect(pastaItems).toHaveLength(20);
expect(menuItems).toHaveLength(162);
```

همچنین این قیمت‌های شاخص را دقیق بررسی کن:

```ts
expect(priceOf("کافه لاته")).toBe(155000);
expect(priceOf("پلاتر خام ۱۶ تکه")).toBe(2290000);
expect(priceOf("سوبایا رول")).toBe(889000);
expect(priceOf("دبل اسپایسی رول")).toBe(959000);
expect(priceOf("پاستا مرغ آلفردو")).toBe(225000);
expect(priceOf("سالاد کرَب و آووکادو")).toBe(610000);
```

تست کن که:

- تمام قیمت‌ها integer مثبت باشند.
- تمام آیتم‌ها `currency: "TOMAN"` داشته باشند.
- IDها تکراری نباشند.
- route برندها تکراری نباشند.
- category هر آیتم در برند مربوطه وجود داشته باشد.
